Level I - Ex 3 - Data Type Conversion

JavaScript is a loose type language, so it allows us to modify variables in flexible ways. It converts automatically data types which which makes variable comparison a lot easier.

When JavaScript performs implicit conversions, the original variable is not modified. The conversion happens only for that specific conversion.

If we have numeric operations the Boolean type is converted to number (1 - true) and (0 - false). In all cases involving Booleans and Strings, both values will be treated as Strings.

Type Casting

Often we cannot rely on JavaScript to perform the right type coercion when working with different kind of data types. There are methods of Object object that can help us with casting:

  • parseInt(val) - Strings are forced to function as integer numbers. For example parseInt("2.35") == 2
  • parseFloat(val) - The same as before except that the value remains a float number with the decimal points. For example parseFloat("2.35") == 2.35
  • Object.toString() - All objects descending from object inherit this method so they can be treated as Strings.
