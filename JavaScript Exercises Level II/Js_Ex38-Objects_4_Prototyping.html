Constructors

Any function you create can serve as a constructor via new keyword so as to make a class definition. class. A constructorâ€™s job is to initialize the object and set any properties needed for an instance of that object. Constructor functions generally should not have return values. The function itself serves as the template for the object, so a return value is redundant. By definition we usually make the first letter of the function uppercase so as to differentiate from regular functions.

Prototyping

In prototypal inheritance you create objects that serve as blueprints for other objects. There is really no formal class definition (at least prior to ES5), so you cannot define a class and make instances of that class but you can use the prototype property and define which members will become part of object instances. The prototype for the object is always kept in memory. That means if the prototype changes at any moment all of the object instances are updated as well.
