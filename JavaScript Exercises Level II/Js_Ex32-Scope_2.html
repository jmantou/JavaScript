Level II - Ex 32 Variable Scope with let & const

The let Keyword

The let keyword for declaring variables was introduced in Javascript ES6. Variables declared inside curly braces ({ }) with 'let' can have block scope and cannot be accessed from outside this block. As we already mentioned varables declared with the var keyword can not have block scope, thus they can be accessed from outside the block. Redeclaring a variable using var can lead to problems because if the variable is inside a block, the variable will be redeclared and outside of this block. We can solve this problem using the let keyword.

The const Keyword

Variables defined with const cannot be redeclared and reassigned. They also have block scope. JavaScript const variables must be assigned a value when they are declared. As a general rule, always declare variables with const unless you know that the value will change.

We very often use const when we want to declare:

  • A new Array
  • A new Object
  • A new Function
  • A new RegExp

NOTE:The keyword const does not exactly defines a constant value, but a constant reference to a value. As a result we can change a constant array and an object defined with const. However we cannot reassigned the array.
